## Тема
Маніпулювання даними SQL у стилі транзакцій (OLTP) для платформи онлайн-курсів.

## Мета
- Набуття практичних навичок роботи з DML-операціями (`SELECT`, `INSERT`, `UPDATE`, `DELETE`).

## 1. Опис структури бази даних
Для виконання роботи використовується база даних, що містить 7 взаємопов'язаних таблиць:
1. `User` — основна таблиця користувачів (супертип).
2. `Student` — дані студентів (підтип User).
3. `Instructor` — дані викладачів (підтип User).
4. `Course` — перелік навчальних курсів.
5. `Module` — навчальні модулі для кожного курсу.
6. `Enrollment` — реєстрації студентів на курси (зв'язок M:N).
7. `Review` — відгуки та оцінки курсів.

## 2. Виконання запитів (DML)

### 2.1. Отримання даних (SELECT)
Використовується для вибору конкретних записів за заданими критеріями.


-- Знайти всіх викладачів з рейтингом вище 4.7
SELECT UserID, Expertise, Rating 
FROM Instructor 
WHERE Rating > 4.7;

-- Отримати список курсів, ціна яких у діапазоні від 400 до 800 грн
SELECT Title, Price 
FROM Course 
WHERE Price BETWEEN 400 AND 800;

## Висновок

У ході виконання лабораторної роботи №3 було успішно реалізовано основні операції маніпулювання даними (DML) в PostgreSQL на базі платформи для онлайн-курсів. Написано та протестовано 7 SELECT-запитів (включно з JOIN, WHERE, GROUP BY та ORDER BY), 5 INSERT-операцій для додавання нових користувачів, студентів, курсів та модулів, 5 UPDATE-операцій для оновлення прогресу, цін та рейтингів, а також 5 DELETE-операцій з дотриманням порядку видалення дочірніх/батьківських записів. У всіх запитах на зміну даних обов'язково використовувалося речення WHERE для точного вибору рядків, а перед кожною модифікацією виконувався SELECT для попередньої перевірки.
